# Data decision register

This register links each data choice in the research extract to its code location, source basis, and paper-use boundary. It is meant to prevent undocumented sample or variable changes as the paper moves from data preparation to estimation.

## Register

| Decision | Code or config | Rule in this repository | Source basis | Paper-use boundary |
| --- | --- | --- | --- | --- |
| Start from the clean upstream IPEDS panel, not raw Access files. | `README.md`; `docs/data_protocol.md`; `scripts/prepare_analysis_panel.py` | This repository treats `panel_clean_analysis_2004_2023.parquet` and `dictionary_lake.parquet` as inputs. Raw NCES database extraction remains in `IPEDSDB_Panel`. | IPEDS-METHOD for the source survey system; upstream repo for construction provenance. | Describe this as the analysis layer. Do not claim this repository rebuilds raw IPEDS. |
| Use a 2009-2023 main COA analysis window. | `docs/data_protocol.md`; `config/analysis_variables.csv`; `src/coa_finaid_subs/prepare_analysis_panel.py` | The build uses `--years 2009:2023` by default because the selected current COA fields begin in 2009 in the verified local panel. | FSA-COA, HEA-472, IPEDS-IC for why COA fields are central; local variable audit for the start year. | State that 2009 is a measurement start, not a policy cutoff. Net-price income-band analyses start later when those fields are observed. |
| Keep four-year Title IV public and private nonprofit institutions in the baseline. | `DEFAULT_MAIN_SECTORS_SPEC = "1,2"`; sample mask in `build_analysis_panel`; `config/analysis_variables.csv` | Baseline rows satisfy `PSET4FLG = 1`, `SECTOR in (1, 2)`, and the selected year window. | IPEDS-METHOD for mandatory Title IV reporting; NCES-TITLEIV-CHANGE for Title IV status and sector-change mechanics. | This defines the baseline estimand. It is not a claim about the full U.S. postsecondary universe. |
| Exclude private for-profit institutions from the baseline, but keep them buildable. | `FORPROFIT_DIAGNOSTIC_SECTORS_SPEC = "3"`; `--include-forprofit-diagnostic`; `docs/replication.md` | For-profits are outside the default outputs and can be built as a diagnostic scope. | CELLINI-GOLDIN; DEMING-GOLDIN-KATZ; IPEDS-FIN-GASB-FASB. | Treat for-profits as an appendix or diagnostic sector unless the paper adds a separate for-profit estimand. |
| Write public and private nonprofit sector files separately. | `DEFAULT_SECTOR_OUTPUT_SPECS`; output directories under `outputs/analysis_panel/` | The default run writes combined baseline, public-only, and private nonprofit-only extracts. | IPEDS-FIN; IPEDS-FIN-GASB-FASB; NCES-DIGEST-INSTITUTIONS. | Main tables should not depend only on a pooled estimate when sector-specific results are feasible. |
| Construct COA totals from tuition and fees, books, room and board, and other expenses. | `add_constructs`; derived fields `COA_*` | COA totals are row sums of the IPEDS charge and allowance fields for the relevant living arrangement. | FSA-COA; HEA-472; IPEDS-IC. | These are institutional COA reports used for financial-aid budgets. They are not student-level realized spending. |
| Define the preferred headroom family before estimation. | `add_constructs`; `config/headroom_measures.csv`; `docs/headroom_measurement.md` | `HEADROOM_MAIN` is the non-tuition part of off-campus not-with-family published COA. `HEADROOM_MAIN_SHARE_COA`, `HEADROOM_MAIN_SHARE_TUITION`, and `LN_HEADROOM_MAIN` are scale checks. | FSA-COA; FSA-PACK; HEA-472; IPEDS-IC. | Use as a measure of the non-tuition budget space inside the published aid cap. Do not describe it as student-level unused aid capacity. |
| Audit headroom components and FTFT-cohort weighting before models. | `scripts/audit_headroom_measures.py`; outputs under `outputs/headroom_measures/` | The audit reports coverage, sector-year trends, correlations, invalid share values, and `SCFA1N`-weighted means for the main measure and component checks. | FSA-COA; IPEDS-IC; local analysis panel audit. | Treat this as measurement validation. It does not estimate aid substitution. |
| Use full-time, first-time aid variables as primary aid outcomes. | `config/analysis_variables.csv`; `add_constructs`; derived fields ending in `_PER_FTFT_COHORT` and grant-share fields | Primary Pell, institutional grant, federal grant, state/local grant, other federal grant, and loan outcomes use the IPEDS FTFT SFA families. | IPEDS-SFA; FSA-PACK. | This matches IPEDS SFA reporting. It does not describe all enrolled undergraduates. |
| Keep all-undergraduate aid fields for checks. | `config/analysis_variables.csv`; derived fields ending in `_PER_UG` | `UAGRNT*`, `UPGRNT*`, and `UFLOAN*` remain in the extract and have audit coverage. | IPEDS-SFA. | Use these as robustness or scale checks when FTFT results need support from broader undergraduate measures. |
| Use `OPENADMP` in the baseline and reserve admit rates/test scores for robustness. | `add_selectivity_constructs`; `audit_variable_config.py`; `docs/selectivity_index.md` | The baseline includes open-admissions status. Admit rate, yield, SAT, and ACT are derived but not required for the main sample. | IPEDS-ADM; DALE-KRUEGER; HOXBY. | Do not use admissions complete cases as the main sample. They exclude open-admissions institutions by design. |
| Build a selective-admissions robustness panel and index. | `analysis_panel_selective_admissions_robustness_*`; `SELECTIVITY_INDEX`; `SELECTIVITY_CATEGORY` | Rows require non-open admissions, a valid admit rate, and at least one usable SAT or ACT midpoint. The index averages within-year admit-rate and test-score z-scores. | IPEDS-ADM; DALE-KRUEGER; HOXBY. | Use the index to compare results among non-open-admissions institutions. Do not market it as a ranking. |
| Use SFA cohort counts as scale controls for now. | `config/analysis_variables.csv`; `LN_SCUGRAD`; `LN_SCFA1N`; audit outputs | Ordinary enrollment and race-share controls do not have usable verified coverage in the local analysis panel, so scale controls use SFA cohort fields with verified coverage. | IPEDS-SFA; local variable audit. | State this as a measurement choice forced by observed coverage. Replace or supplement it if enrollment and race-share fields are later populated. |
| Exclude ordinary enrollment and race-share controls from the baseline. | `docs/variable_selection.md`; `docs/data_protocol.md` | `ENRTOT`, `FTE`, and `PCTENR*` are not used as baseline controls because the verified local panel does not provide usable coverage for this analysis sample. | Local parquet and variable-coverage audit. | Describe this as a data limitation. If coverage is recovered later, add those controls as a named sensitivity. |
| Harmonize finance controls by sector. | `FINANCE_DEFS`; `sector_finance_value`; derived `FIN_*` fields | Public rows use `F1*`, private nonprofit rows use `F2*`, and for-profit diagnostic rows use `F3*` where available. NCES notes that a small number of public institutions can report on FASB forms, so finance coverage should be checked before final estimation. | IPEDS-FIN; IPEDS-FIN-GASB-FASB. | Cross-sector finance controls should be described as harmonized approximations, not identical accounting concepts. |
| Keep public state and local appropriations as a public-sector measure. | `FIN_STATE_LOCAL_APPROPS_PUBLIC` | Public state/local appropriations and grants are populated for public institutions only. | IPEDS-FIN; IPEDS-FIN-GASB-FASB. | Use this in public-sector models or interactions. Do not force it into private nonprofit specifications as a common control. |
| Treat current income-band net price as secondary. | `HARMONIZED_NET_PRICE_DEFS`; `NET_PRICE_*`; `NET_PRICE_*_NEGATIVE_FLAG`; `NET_PRICE_*_NONNEG` | Public rows use `NPIS*`; private rows use `NPT*`. Raw values are retained. Nonnegative copies set negative values missing and keep flags. | IPEDS-NETPRICE; NCES-DIGEST-NETPRICE; FSA-PACK. | Net price is a diagnostic outcome because it has a thinner sample and a different student denominator than the main COA and aid fields. |
| Retain raw negative net-price values and create nonnegative analysis copies. | `add_constructs`; `NET_PRICE_*_NEGATIVE_FLAG`; `NET_PRICE_*_NONNEG` | The extract never overwrites raw net-price values. It flags negative values and writes clean copies for analyses that require nonnegative prices. | IPEDS-NETPRICE; NCES-DIGEST-NETPRICE; local audit. | Do not delete negative raw values before investigating them. Use flags for sensitivity and reporting. |
| Audit zero values in aid fields. | `aid_zero_consistency_rows`; `analysis_aid_zero_consistency.csv`; `analysis_aid_zero_suspect_rows.csv` | The build checks each aid family across count, percent, average, total, and denominator fields. True zeros require zero count, percent, and total with no positive average signal. Suspect zeros are written separately when zero totals conflict with positive counts, percentages, or averages. | IPEDS-SFA; local audit. | Keep true zeros in analysis. Inspect suspect zeros before treating them as substantive zeros. |
| Audit distribution tails before winsorization. | `scripts/audit_extremes.py`; `extreme_audit_variable_profile.csv`; `extreme_audit_review_candidates.csv`; `extreme_audit_top_rows.csv` | The audit profiles every variable, records type and missingness, computes numeric quantiles and IQR fences, writes top and bottom rows, and lists variables needing review. | Local analysis panel audit. | Do not winsorize by default. Add caps or trims later only as variable-specific, documented sensitivity rules. |
| Report descriptive statistics before and after display-only winsorization. | `config/descstat_variables.csv`; `scripts/build_descstat_tables.py`; `notebooks/01_descstat_pre_post_winsorization.ipynb` | The table builder computes raw statistics and capped statistics from the same panel, writes paper and appendix tables, and leaves the parquet unchanged. | Local analysis panel audit. | Use the capped columns to show tail sensitivity in descriptive exhibits. Do not treat this as a change to the baseline panel. |
| Lock the first research design before estimation. | `docs/research_design.md`; `config/model_specifications.csv`; `scripts/audit_model_plan.py` | The design memo fixes the baseline estimand, outcomes, controls, evidence ladder, and claim boundaries. The model-plan audit checks planned model variables and complete-case counts before any regression is run. | FSA-COA; FSA-PACK; IPEDS-SFA; local analysis panel audit. | Treat this as a pre-estimation plan. Any causal design using aid shocks still requires a verified policy-shock file. |
| Retain raw IPEDS metadata fields and derive conservative exposure flags. | `METADATA_COMPONENTS`; `derive_metadata_flags`; metadata audit tables | Raw `IMP_*`, `LOCK_*`, `REV_*`, `IDX_*`, `PRCH_*`, and parent-child fields remain. Derived flags summarize imputation, revision, and parent-linked reporting. | IPEDS-METHOD; IPEDS-MANUAL. | Use the flags for sample description and sensitivity checks. They do not replace the raw IPEDS status codes. |
| Treat the panel as intentionally unbalanced. | `build_panel_balance_by_institution`; `analysis_min_years_sensitivity.csv`; `docs/sample_dynamics.md` | The baseline keeps institutions that satisfy the sample rule in any year, then reports balance and minimum-years sensitivity. | NCES-TITLEIV-CHANGE; NCES-DIGEST-INSTITUTIONS; local balance tables. | Describe entries and exits as sample entry and exit unless separate evidence verifies openings, closures, or mergers. |
| Classify entry and exit reasons using the full panel. | `entry_exit_reason_audit`; `analysis_entry_exit_reason_audit.csv` | Each entrant or exiter is classified as full-panel first appearance, full-panel disappearance, `PSET4FLG` transition, `SECTOR` transition, both, or other gap. | NCES-TITLEIV-CHANGE; IPEDS-METHOD. | Use this table to separate reporting/sample-rule changes from real institutional turnover. |
| Run complete-case audits by empirical scenario. | `audit_variable_config.py`; outputs under `outputs/variable_audit/` | The audit reports column presence, coverage, and complete-case counts for primary, finance, net-price, metadata, and admissions scenarios. | Local parquet and dictionary audit. | Treat complete-case counts as reproducibility evidence. Do not call them literature facts. |
| Enforce row identity and file provenance checks. | `check_unique_key`; `sha256_file`; build summary JSON | The script checks one row per `UNITID` and year, no missing identifiers, and input/output hashes. | Replication practice; local code. | Report hashes in replication notes when a result depends on a specific input panel. |

## Sources still needing paper-level treatment

The register documents the data layer. The paper will still need a separate literature review for the economic mechanism: institutional grant substitution, tuition discounting, and how Title IV caps may shape pricing or aid packaging. That literature should not be buried in this data register. It belongs in the paper's theory and identification sections.
